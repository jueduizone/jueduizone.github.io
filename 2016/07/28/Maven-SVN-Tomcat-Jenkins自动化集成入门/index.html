Maven+SVN+Tomcat+Jenkins自动化集成入门

1. 安装与技巧

本文的安装是基于Mac，使用官方的.pkg安装包进行安装，Jenkins版本为2.7.1，下载地址:点击下载。
安装完成后，会自动打开浏览器，进入Getting Started界面：
1. Unlock Jenkins，即输入Jenkins安装目录内的/secrets/initialAdminPassword文件内的密码即可。
2. Customize Jenkins，主要是进行Plugins的安装，除了本身推荐的，这边还推荐一个很好用的Publish Over SSH，主要功能是将包文件上传到服务器，后面会用到。
选择完插件后点击Install进行安装即可。
3. Create First Admin User，即创建第一个管理员用户，输入用户名、密码等所需信息，然后点击保存开始，安装就基本Mission Complete了。
常用命令：
更改启动端口：
sudo defaults write /Library/Preferences/org.jenkins-ci httpPort 8090
Jenkins 启动方法：
sudo launchctl load /Library/LaunchDaemons/org.jenkins-ci.plist
Jenkins 关闭方法：
sudo launchctl unload /Library/LaunchDaemons/org.jenkins-ci.plist
Jenkins卸载方法：
/Library/Application Support/Jenkins/Uninstall.command

2. 基本配置

这里主要配置JDK、Maven、SSH环境或信息。

  • JDK和Maven的配置在左侧菜单的[系统管理 - Global Tool Configuration]里面，进入后可以看到JDK和Maven选项。
    点击JDK安装，点击取消自动安装，JAVA_HOME中填入本地的的JDK安装目录即可。
    点击Maven安装，点击取消自动安装，MAVEN_HOME中填入本地的Maven安装目录即可。

  • SSH配置在[系统管理 - 系统设置]里面，进入后底部有一个Publish over SSH（这里采用密码登录方式，不使用证书），点击SSH Servers旁边的增加，输入服务器的地址，用户名（非root用户需要开启ftp等相应权限，否则会出现没有权限上传文件的错误）等信息，然后点击下方的高级按钮，选择密码认证的方式，输入密码即可，可以用Test按钮进行测试。

3. 新建集成任务

  • 点击左侧菜单的新建或者主页面的创建一个新任务，进入新建页面，填写项目名称，选择下方的构建一个自由风格的软件项目然后点击下方的OK按钮进入下一步配置。
  • 进入具体的配置页面，按照集成流程共有[General->源码管理->构建触发器->构建环境->构建->构建后操作]六个配置选项，这里主要配置源码管理、构建、构建后操作。
    1. 源码管理:
    根据版本管理工具选择不同的方式，这里选择的是SVN进行管理，其中Credentials 配置的是svn的用户名密码，Local module directory 配置的是从版本库拉下的代码存放的位置，默认是.，为Jenkins默认存放的根目录。

    2. 构建:
    这里选择构建的方式，因为是基于Maven的，所以选择Invoke top-level Maven targets方式，然后填入Maven打包命令，填入pom文件名称即可。


    然后再选择增加构建步骤，选择Execute Shell，这里填入Shell命令，主要对构建好的包进行打包压缩，便于上传。

    2. 构建后操作:
    这步操作主要将打包文件上传到服务器，并解压运行，点击增加构建后操作步骤，选择Send build artifacts over SSH，就是前面说到的Publish Over SSH插件，主要填写项目如下:
    Name选项可以选择前面配置过的服务器信息；
    Source files填入需要上传的文件；
    Remove prefix移除不需要的前缀，这里需要注意一下上面的Source files如果包含了路径，则上传后，它会在指定服务器路径下新建该路径存放文件，如Source files填写的为/targe/test.war，服务器指定的路径为/opt，则上传后存放的路径为/opt/target/test.war；
    Remove prefix配置/target可以移除该路径，直接将文件存放在指定的目录下/opt/test.war），
    Remote directory上传到的服务器的目录，因为我前面在SSH的配置指定了基础目录为部署目录，所以这里不需要再填写；
    Exec command执行的Shell命令。

    因为我这里是将项目(war包形式)部署在tomcat中，同时tomcat开启了war包自动部署，所以这里只要将压缩包上传到部署目录解压即可，如果没有开启自动部署或者其它类型的项目，则可能还需要编写Shell命令进行相应容器服务的重启，因为都是常规的命令，所以不再详述。
    以上全部配置好，点击保存即可，回到首页就可以看到刚刚配置的项目，点击最后的绿色播放按钮就可以开始构建。

4. 开始构建

开始构建后，可以在左侧的构建执行状态中看到执行的进度。

点击后面的构建次数，进入本次的构建详情，然后点击左侧菜单的Console Output可以看出构建的具体日志输出，如果构建有问题，可以在这里看出具体问题原因，在哪一步出了问题。


如果构建成功，可以看到首页项目列表中最前面的圆球图标为蓝色，如果失败则为红色，这时候就需要去日志输出里面看原因了。
