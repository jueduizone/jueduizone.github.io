Jenkins使用进阶（持续更新）

  • Jenkins
  • 技术

本文主要记录在使用Jenkins的工作中遇到的一些问题或者需求，并提供相应的解决方案（全部测试通过），因为是刚刚学习使用Jenkins，有些问题在网上并没有找到合适自己的解决方法，是自己探索出来的，所以可能并不是很好的方式，但至少能解决当下的问题，如果大家有更好的解决方案，恳请指教。目前涉及的内容主要如下：

  1. 构建结果邮件发送
  2. 配置文件替换

1. 构建结果邮件发送

这个主要实现的是构建完成后，像相关人员发送一份推送邮件，告知构建信息及结果。这里主要使用到两个插件：Email Extension Plugin、Email Extension Template Plugin。

  1. 先设置邮件的发送账户(SMTP)信息。进入[系统管理-系统设置-Extended E-mail Notification]，主要设置选项：
    SMTP server：设置邮件提供商的SMTP地址，例如163企业邮箱的是smtp.ym.163.com。
    Default Content Type：内容展现的格式，一般选择HTML。
    Default Recipients：默认收件人。
    Use SMTP Authentication：使用SMTP身份验证，点开Default user E-mail suffix下面的高级按钮才能看到，选中。
    User Name：邮件发送账户的用户名。
    Password：邮件发送账户的密码。
    SMTP port：SMTP端口。

  2. 设置邮件模板，模板的功能主要是方便后面直接在相应的工程里，不需要再重复编写标题、内容等共通信息。进入[系统管理—Editable Email Notification Templates]，会首先看到已有的模板列表，可以进行编辑、删除操作，然后左侧菜单的Add New Template可以新增邮件模板，主要设置选项：
    Name：模板名称，工程中会选择相应名称的模板调用。
    Default Subject：邮件标题，这里提供一个模板：构建通知:$PROJECT_NAME - Build # $BUILD_NUMBER - $BUILD_STATUS!。
    Default Content：邮件内容。这里提供一个模板：

    Attachments：邮件附件。
    Attach Build Log：是否将构建日志作为附件，三个选项分别为需要，不需要，压缩作附件。
    Triggers：邮件发送触发条件，需要点击下面的Advanced Setting按钮才会出来。然后点击Add Trigger按钮，选择触发事件，我这里选择Always，即不管构建状态，均发送邮件。然后还有一个Send To选项，我这里分别选择Developers和Recipient List，即发送给构建的人和邮件列表中的人员。

    全部设置完成后，点击提交按钮即可。

  3. 最后给相应的工程绑定邮件模板，进入工程的构建配置，选择[构建后操作-增加构建后操作步骤]，选择Editable Email Notification Templates，然后在Templates中的下拉列表中选择刚刚设置的模板即可。

2. 配置文件替换

一般工程至少都有测试环境和正式环境，不同环境的配置文件是不一样的，配置文件替换功能就是事先在Jenkins上设置好正确的配置文件，构建的时候替换相应的文件，这样就可以避免每次上传代码的时候都要记住改环境或者手动的改环境，降低环境出错的概率。这里需要一个安装插件Config File Provider Plugin。

  1. 新建配置文件，插件安装好，在系统管理里可以找到Managed files选项，进入后，可以看到配置过的文件列表，可以进行编辑或者删除。

    在左侧菜单有一个Add a new Config菜单，进入可以新增配置文件，它会让你选择新建的配置文件类型，根据要求选择，我这里选择的是自定义文件。

    进入具体设置页面，主要设置Name、Comment、Content三个选项，Name就是配置文件的名字，Comment备注，Content就是配置文件的具体内容。
  2. 工程配置，配置文件新建好后，进入工程的构建设置，找到构建环境模块，可以看到Provide Configuration files选项，选择后，主要需要设置File、Target两个选项，File就是选择刚刚新建的配置文件，Target是选择需要被替换的文件，可以包含路径，这里的默认路径是当前工程的跟目录。

    设置完提交即可，重新构建时可以看到在构建前Jenkins会替换相关文件。
